# MindAtlas 系统说明

> 版本：1.0
> 更新日期：2026年1月

---

## 一、系统简介

**MindAtlas**（思维图谱）是一款**个人知识与经历管理系统**，旨在帮助用户系统性地记录、关联、回顾和总结个人的知识积累与人生经历。

系统的核心理念是：**让碎片化的记忆变成结构化的资产**。

通过 MindAtlas，你可以：
- 记录学习笔记、项目经历、比赛成果、技能掌握等各类信息
- 建立知识点之间的关联关系，形成个人知识图谱
- 按时间轴回顾人生历程
- 借助 AI 能力自动生成摘要和标签
- 随时检索和回答"我做过什么"、"我擅长什么"

---

## 二、背景与定位

### 2.1 为什么需要 MindAtlas？

在日常生活和工作中，我们积累了大量的知识、经验和成果：

- 学习过的技术和概念
- 参与过的项目和比赛
- 获得的证书和荣誉
- 重要的人生经历和里程碑

然而，这些宝贵的信息往往散落在各处——笔记软件、博客、云盘、聊天记录中，形成了**信息孤岛**。

### 2.2 现有工具的痛点

| 工具类型 | 存在的问题 |
|----------|------------|
| 笔记软件 | 信息孤立，难以建立关联 |
| 博客平台 | 偏向展示，不便于个人回顾 |
| 云盘存储 | 只是文件堆积，缺乏结构 |
| 简历/作品集 | 静态展示，无法动态更新 |

### 2.3 MindAtlas 的定位

MindAtlas 定位为**个人数字大脑**，核心价值在于：

1. **统一管理** - 一个平台记录所有类型的知识和经历
2. **关联发现** - 自动或手动建立信息之间的联系
3. **时间维度** - 按时间轴展示个人成长历程
4. **智能增强** - AI 辅助生成摘要、标签和洞察
5. **数据自主** - 数据完全属于用户，可导出可迁移

## 三、核心概念

### 3.1 Entry（记录）

**Entry** 是系统中的最小信息单元，代表一条知识或经历记录。

每条 Entry 包含：
- **标题** - 简洁描述这条记录
- **内容** - 详细的 Markdown 格式描述
- **类型** - 知识、项目、比赛、经历等
- **时间** - 发生的时间点或时间段
- **标签** - 用于分类和检索
- **关联** - 与其他 Entry 的关系
- **附件** - 相关的文件和图片

### 3.2 类型系统

系统预置了常用的记录类型，用户也可以自定义：

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| 知识 (Knowledge) | 学习笔记、概念理解 | 技术学习、读书笔记 |
| 项目 (Project) | 个人或工作项目 | 开发项目、创业项目 |
| 比赛 (Competition) | 竞赛、黑客松 | 编程比赛、创新大赛 |
| 经历 (Experience) | 人生经历、重要事件 | 旅行、工作经历 |
| 成果 (Achievement) | 荣誉、证书、奖项 | 获奖、认证 |
| 技术 (Technology) | 技能、工具掌握 | 编程语言、框架 |
| 资料 (Document) | 参考文档、收藏 | 重要文档、学习资料 |

### 3.3 关系网络

Entry 之间可以建立多种类型的关联关系：

- **属于** (belongs_to) - 表示从属关系
- **使用** (uses) - 表示使用了某项技术或工具
- **参与** (participates) - 表示参与了某个活动
- **相关** (related_to) - 表示一般性关联
- **衍生** (derived_from) - 表示基于某个基础发展而来

## 四、功能模块

### 4.1 记录管理

**核心功能：创建、编辑、删除、搜索记录**

- 支持 Markdown 格式编辑
- 灵活的时间设置（无时间/时间点/时间段）
- 多标签分类
- 全文搜索和筛选
- 分页浏览

### 4.2 知识图谱

**核心功能：可视化展示记录之间的关联关系**

- 力导向图布局
- 按类型着色区分
- 点击节点查看详情
- 支持缩放和拖拽

### 4.3 时间轴

**核心功能：按时间顺序展示记录**

- 按年/月分组展示
- 支持类型筛选
- 直观的时间线视图
- 快速定位历史记录

### 4.4 附件管理

**核心功能：管理记录相关的文件**

- 支持多种文件格式
- 文件上传和下载
- 与记录关联存储
- 安全的对象存储

### 4.5 标签系统

**核心功能：灵活的分类和检索**

- 自定义标签颜色
- 标签统计
- 按标签筛选记录
- 支持多标签组合

### 4.6 类型配置

**核心功能：自定义记录类型**

- 添加/编辑/删除类型
- 设置类型颜色和图标
- 类型使用统计

### 4.7 AI 服务配置

**核心功能：配置 AI 服务提供商**

- 支持多个 AI 服务商
- API 密钥安全存储
- 自动获取可用模型列表
- 连接测试功能
- 一键切换激活的服务商

### 4.8 多语言支持

**核心功能：中英文界面切换**

- 支持简体中文和英文
- 一键切换语言
- 语言偏好自动保存

> - 在 MindAtlas 中浏览自己的所有记录

## 八、设计理念

### 8.1 简单优先

系统采用单体架构，避免过度设计。功能实现以解决实际问题为导向，不追求技术复杂度。

### 8.2 数据自主

用户的数据完全属于用户自己：
- 数据存储在用户可控的数据库中
- 支持完整数据导出
- 不依赖特定云服务

### 8.3 AI 增强而非依赖

AI 是辅助工具，而非核心依赖：
- 系统在没有 AI 的情况下完全可用
- AI 功能是锦上添花，提升效率
- 用户始终保持对内容的控制权

### 8.4 渐进式演进

系统设计支持渐进式升级：
- 核心功能稳定可靠
- 新功能可以逐步添加
- 技术栈可以平滑迁移
