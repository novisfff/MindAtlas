## 系统概述

MindAtlas 是一个个人知识与经历管理系统，帮助你：

- **记录**：创建结构化的知识和经历条目
- **关联**：建立条目之间的关系网络
- **搜索**：快速检索历史记录
- **分析**：通过图谱可视化知识结构
- **总结**：借助 AI 助手整理和回顾

### 核心概念

| 概念 | 说明 |
|------|------|
| **Entry** | 系统核心实体，代表一条知识或经历记录 |
| **类型 (Type)** | Entry 的分类，如"知识"、"项目"、"竞赛"等 |
| **标签 (Tag)** | 灵活的标记系统，用于多维度分类 |
| **关系 (Relation)** | Entry 之间的关联，如"属于"、"使用"、"相关"等 |
| **附件 (Attachment)** | 关联到 Entry 的文件 |

## 仪表盘

**路径**: `/dashboard`

仪表盘是系统首页，提供数据概览：

- **快捷操作**：快速创建新 Entry 的入口

## Entry 管理

**路径**: `/entries`

Entry 是系统的核心功能，用于记录和管理你的知识与经历。

### 浏览 Entry 列表

1. 点击侧边栏的「条目」进入列表页
2. 列表显示所有 Entry，包含标题、类型、标签、创建时间
3. 点击任意 Entry 可查看详情

**筛选功能**：
- 按类型筛选：选择特定类型查看
- 按标签筛选：选择一个或多个标签
- 关键词搜索：在搜索框输入关键词

### 创建新 Entry

1. 点击列表页右上角的「新建」按钮
2. 填写 Entry 信息：

| 字段 | 说明 | 必填 |
|------|------|------|
| 标题 | Entry 的名称 | 是 |
| 类型 | 选择预设的类型 | 是 |
| 内容 | 支持 Markdown 格式 | 否 |
| 标签 | 选择或创建标签 | 否 |
| 时间模式 | NONE/POINT/RANGE | 否 |

3. 点击「保存」完成创建

### 编辑 Entry

1. 在详情页点击「编辑」按钮
2. 修改需要更新的字段
3. 点击「保存」提交更改

**管理关系**：
- 在详情页的「关系」区域添加与其他 Entry 的关联
- 支持的关系类型：belongs_to、uses、related_to 等

**管理附件**：
- 在详情页的「附件」区域上传文件
- 支持拖拽上传或点击选择文件
- 文件存储在 MinIO 对象存储中

### 搜索 Entry

1. 使用列表页顶部的搜索框
2. 输入关键词后按回车或点击搜索图标
3. 搜索范围包括标题和内容

## 知识图谱

**路径**: `/graph`

知识图谱以可视化方式展示 Entry 之间的关联网络。

### 基本操作

- **缩放**：滚动鼠标滚轮放大/缩小视图
- **拖拽**：按住空白区域拖动画布
- **选中节点**：点击节点查看 Entry 详情
- **移动节点**：拖拽节点调整位置

### 图谱元素

| 元素 | 说明 |
|------|------|
| 节点 | 代表一个 Entry，颜色对应类型 |
| 连线 | 代表 Entry 之间的关系 |
| 标签 | 显示 Entry 标题 |

## 时间线

**路径**: `/timeline`

时间线按时间顺序展示 Entry，适合回顾经历和追踪进展。

## AI 助手

**路径**: `/assistant`

AI 助手基于 LangChain 构建，支持自然语言交互和工具调用。

### 基本对话

1. 点击侧边栏的「助手」进入对话页面
2. 在输入框输入问题或指令
3. AI 会根据意图自动选择合适的技能处理

### 内置技能

| 技能 | 说明 | 示例 |
|------|------|------|
| 搜索记录 | 搜索 Entry | "帮我找一下关于 React 的笔记" |
| 查看详情 | 获取 Entry 详细信息 | "看看这条记录的具体内容" |
| 快速统计 | 获取系统统计数据 | "我有多少条记录？" |
| 智能记录 | 快速创建 Entry | "帮我记录一下今天学的 Docker 知识" |
| 知识综合 | 整理归纳知识 | "总结一下关于微服务的笔记" |

### 对话管理

- **新建对话**：点击左侧「新对话」按钮
- **历史对话**：左侧列表显示历史对话记录
- **删除对话**：悬停对话项，点击删除图标

### 前置条件

使用 AI 助手前，需要先在设置中配置 AI 服务商。

## 系统设置

**路径**: `/settings`

系统设置页面提供各项配置功能的入口。

### Entry 类型管理

**路径**: `/settings/entry-types`

管理 Entry 的分类类型。

**操作步骤**：
1. 点击「新建类型」添加新类型
2. 填写类型名称、选择图标和颜色
3. 点击保存完成创建

**编辑/删除**：
- 点击类型卡片进入编辑
- 删除类型前需确保没有 Entry 使用该类型

### 标签管理

**路径**: `/settings/tags`

管理系统中的标签。

**操作步骤**：
1. 点击「新建标签」
2. 输入标签名称
3. 点击保存

**批量操作**：
- 支持合并重复标签
- 支持批量删除未使用的标签

### AI 服务商配置

**路径**: `/settings/ai-providers`

配置 AI 服务商以启用 AI 功能。

**配置步骤**：
1. 点击「添加服务商」
2. 填写配置信息：

| 字段 | 说明 |
|------|------|
| 名称 | 服务商显示名称 |
| API Key | 服务商提供的密钥 |
| Base URL | API 地址（如 OpenAI 兼容接口） |
| 模型 | 使用的模型名称 |

3. 点击保存，系统会加密存储 API Key
4. 点击「激活」启用该服务商

### 助手工具配置

**路径**: `/settings/assistant-tools`

配置 AI 助手可调用的工具。

**内置工具**：
- search_entries - 搜索 Entry
- get_entry_detail - 获取 Entry 详情
- create_entry - 创建 Entry
- get_statistics - 获取统计数据
- list_entry_types - 列出类型
- list_tags - 列出标签

**自定义工具**：
支持添加远程 API 作为工具，配置 HTTP 端点和参数。

### 助手技能配置

**路径**: `/settings/assistant-skills`

技能是预定义的工作流，AI 根据用户意图自动选择执行。

**技能模式**：
- **Agent 模式**：AI 自主决定调用哪些工具
- **Pipeline 模式**：按预设步骤顺序执行

**自定义技能**：
1. 点击「新建技能」
2. 配置技能名称、描述、意图示例
3. 选择可用工具
4. 设置执行步骤（Pipeline 模式）

## 国际化

系统支持中文和英文界面。

### 切换语言

1. 点击页面右上角的语言切换按钮
2. 选择「中文」或「English」
3. 界面立即切换为所选语言

语言偏好会自动保存，下次访问时保持设置。
